# Workflow (Git + Branching)

## Branch naming
Use:
- feature/<name>-<topic>
Examples:
- feature/fadi-architecture
- feature/ziming-camera
- feature/jichao-tests

## Rules
- Never push directly to `main`
- All changes go via Pull Request (PR)
- At least 1 teammate must review the PR before merge
- Keep PRs small and focused (one feature / fix per PR)

## Commit messages
Use short, descriptive commits:
- "Add thread-safe queue"
- "Implement sun centroid detection"
- "Add latency logging timestamps"
